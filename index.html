  • Fuzzi Moss
    • A still-developed offshoot from my honours dissertation. This is a library for introucing faults into sociotechnical models.
    • This model is composed of two parts:
      • A sociotechnical modelling system, in raw python, that focuses on workflows in a pragmatic way
      • A mutation library that intercepts python function execution, recompiles the source code with safe and infinitely powerful alterations, and executes a slightly altered version of the mutant.
        This has the effect of running a different version of a function every time the funtion is executed, something I haven't found in any other mutation library.
    • This is still under active development, with Tim Storer at Glagow university. We're finishing up a paper on what's been done so far, and it'll be submitted to conferences soon. Watch this space!

  • Music Organiser
    • A script to fix the shortcomings I found in a solution to a similar problem by @bamos on GitHub.
      Organises music based on a handful of tags, by searching files for metadata, and produces a filesystem structure defined by the user.

      For example, you might want your music library reorganised so that tracks which are numbered are put into albums, which are put into folders for each artist, which are then sorted by genre. The parameter --struture=genre/artist/album/track+title would do the trick.
      This is still under active development. Feel free to contribute!
